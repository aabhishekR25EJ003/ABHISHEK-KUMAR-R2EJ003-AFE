Technical Model Report: NVIDIA LocateAnything-3B

An architectural and training breakdown of NVIDIA’s premier 3-billion-parameter vision-language spatial localization model[cite: 2].

1. Executive Summary

The LocateAnything-3B model is a state-of-the-art, 3-billion-parameter generalist Vision-Language Model (VLM) developed by NVIDIA[cite: 2]. It is specifically optimized for high-speed, high-quality visual grounding and spatial localization tasks[cite: 2].

Unlike traditional VLMs that process image understanding abstractly or output coordinates sequentially, LocateAnything-3B directly bridges natural language instructions with precise spatial coordinate outputs[cite: 2]. It handles open-vocabulary object detection, phrase grounding, Optical Character Recognition (OCR) text localization, Graphical User Interface (GUI) element grounding, and point-based targeting[cite: 2].

2. Core Architecture & Technical Innovations

Component Breakdown

LocateAnything-3B operates natively at high resolution and integrates three main architectural components[cite: 2]:

  • Vision Encoder: MoonViT-SO-400M (MIT License) – Extracts high-fidelity visual tokens directly at native resolutions (up to 2.5K) to capture fine-grained spatial and textual details[cite: 2].
  • Language Decoder: Qwen2.5-3B-Instruct (Qwen Research License) – Serves as the language reasoning foundation, extending prompt context lengths up to 24K tokens[cite: 2].
  • Multimodal Projector: A multi-layer perceptron (MLP) mapping layer that aligns visual features into the text token space[cite: 2].

The Innovation: Parallel Box Decoding (PBD)

Traditional localization models rely on Autoregressive Decoding (predicting coordinates sequentially token-by-token)[cite: 2]. This creates a structural bottleneck and ruins geometric coherence[cite: 2]. LocateAnything-3B introduces Parallel Box Decoding (PBD), a block-wise multi-token prediction (MTP) framework that treats an entire bounding box or target point as an atomic, fixed-length block unit[cite: 2].

Traditional Approach: (x1) → (y1) → (x2) → (y2) [Sequential Bottleneck]
NVIDIA PBD Approach: [x1, y1, x2, y2] Parallel Token Block Pass
  • Throughput: PBD predicts complete bounding box structures in a single parallel forward pass step, yielding up to 2.5× higher throughput than previous quantized coordinate decoders and up to 10× faster speeds than text-coordinate decoders (e.g., Qwen3-VL)[cite: 2].
  • Decoding Modes:
    1. Fast Mode (MTP): Predicts full boxes entirely in parallel for ultra-high throughput[cite: 2].
    2. Slow Mode (NTP): Reverts to autoregressive Next-Token Prediction for strict sequence verification[cite: 2].
    3. Hybrid Mode: The default setting. It leverages Fast Mode for execution speed and seamlessly falls back to Slow Mode only if structural ambiguity or formatting irregularities are encountered[cite: 2].

3. Training Dataset: LocateAnything-Data

NVIDIA engineered a massive, scalable data compilation containing over 138 Million language queries and 785 Million bounding boxes spanning multiple operational domains[cite: 2]:

Task Domain Dataset Composition % (Queries) Focus Areas & Features
General Object Detection 66.9% Dense coordinate alignments, standard natural scenes (COCO, LVIS), multi-object detection[cite: 2].
GUI Element Grounding 16.5% Graphical User Interface layout mapping, bounding icons/buttons for computer use & digital agents[cite: 2].
Referring Comprehension 7.3% Complex relational natural language inputs matched to a target image region (e.g., "the second cup from the left")[cite: 2].
Text Localization (OCR) 3.6% Reading and tightly grounding structured textual strings inside dynamic or complex documents[cite: 2].
Layout Grounding 3.5% Enriches structural geometric reasoning for document parsing and structural charts[cite: 2].
Point-Based Localization 2.2% Fine-grained single-coordinate pointing (e.g., center of gravity or specific keypoint targets)[cite: 2].

Labeling Methodology

The corpus was curated using a hybrid approach combining original human-annotated open-source datasets with high-fidelity, model-assisted automated annotations[cite: 2]. Generative labeling pipelines utilized foundational vision agents (Qwen3-VL, Molmo, SAM 3, and Rex-Omni) alongside custom automated post-verification steps to minimize geometric format errors[cite: 2].

4. Training Parameters & Pipeline

LocateAnything-3B employs a specialized four-stage fine-tuning pipeline to step up model understanding from generalized visual tokens to strict, block-aligned coordinate values[cite: 2].

  • Stage 1: Multimodal Knowledge Adaptation: Standard image captioning, basic Visual Question Answering (VQA), and general visual grounding to establish standard VLM cross-attention features[cite: 2].
  • Stage 2 & 3: Grounding & Coordinate Alignment: Injecting coordinate block targets into sequence formulations[cite: 2]. The model trains sequence lengths up to a maximum of 25,600 tokens to map dense object screens accurately[cite: 2].
  • Stage 4: Joint Dual-Formulation Fine-Tuning: Training objective parameters simultaneously optimize both parallel block targets and localized autoregressive error recovery, allowing the execution of the dynamic Hybrid Inference Mode[cite: 2].

5. Intended Use Cases & Limitations

Ideal Applications

  • Automated Dataset Labeling: Automating bounding box generation for custom object detection models (e.g., exporting immediate COCO/YOLO datasets without manual labeling workflows)[cite: 2].
  • Physical AI & Robotics: Giving real-world embedded physical systems a spatial baseline via natural language prompts[cite: 2].
  • GUI & Web Agents: Empowering automated desktop workflows to visually capture and interact with web layouts, UI screens, and mobile dashboards[cite: 2].

Model Limitations

  • License Constraints: The model is released under the NVIDIA Non-Commercial Research License and is restricted from direct production monetization workflows[cite: 2].
  • Hardware Overhead: Requires structured framework execution environments[cite: 2]. While inference fits in lightweight consumer rigs via 4-bit NF4 quantization (~3.5 GB VRAM), executing custom local adjustments natively demands intensive VRAM pipelines to sustain the 25.6K max sequence context configurations[cite: 2].
